Setting up API Gateway

API Gateway Configuration for E-commerce Microservices

In this section, we’ll configure Amazon API Gateway as the single entry point for our e-commerce microservices system. The API Gateway will route requests to the appropriate microservices and provide centralized authentication, authorization, and monitoring.

Architecture Overview

Our API Gateway will handle requests for all microservices:

Client → API Gateway → [Supplier Service | Consumer Service | Inventory Service | Product Service | Payment Service | Order Service]

API Gateway Routes

1. Supplier Service Routes

  • GET /api/v1/suppliers - List all suppliers
  • GET /api/v1/suppliers/{id} - Get supplier details
  • POST /api/v1/suppliers - Create new supplier
  • PUT /api/v1/suppliers/{id} - Update supplier
  • DELETE /api/v1/suppliers/{id} - Delete supplier

2. Consumer Service Routes

  • GET /api/v1/consumers - List consumers
  • GET /api/v1/consumers/{id} - Get consumer profile
  • POST /api/v1/consumers - Register new consumer
  • PUT /api/v1/consumers/{id} - Update consumer profile

3. Product Service Routes

  • GET /api/v1/products - List products with pagination
  • GET /api/v1/products/{id} - Get product details
  • GET /api/v1/products/category/{category} - Get products by category
  • POST /api/v1/products - Create new product (admin only)

4. Inventory Service Routes

  • GET /api/v1/inventory/{productId} - Check product availability
  • POST /api/v1/inventory/reserve - Reserve inventory
  • POST /api/v1/inventory/release - Release reserved inventory

5. Order Service Routes

  • GET /api/v1/orders - Get user orders
  • GET /api/v1/orders/{id} - Get order details
  • POST /api/v1/orders - Create new order
  • PUT /api/v1/orders/{id}/status - Update order status

6. Payment Service Routes

  • POST /api/v1/payments - Process payment
  • GET /api/v1/payments/{id} - Get payment status
  • POST /api/v1/payments/{id}/refund - Process refund

Implementation Steps

Step 1: Create API Gateway

  1. Go to API Gateway Console
  2. Click Create API
  3. Choose REST API → Build
  4. Set API name: ecommerce-microservices-api
  5. Set Description: API Gateway for E-commerce Microservices
  6. Click Create API

Step 2: Configure CORS

{
  "origin": "*",
  "methods": ["GET", "POST", "PUT", "DELETE", "OPTIONS"],
  "headers": ["Content-Type", "Authorization", "X-Amz-Date", "X-Api-Key"]
}

Step 3: Create Resource Structure

Create the following resource hierarchy:

/
├── api
    └── v1
        ├── suppliers
        ├── consumers  
        ├── products
        ├── inventory
        ├── orders
        └── payments

Step 4: Configure Service Integration

For each service, configure:

  • Integration Type: HTTP Proxy
  • HTTP Method: ANY
  • Endpoint URL: http://{service-load-balancer-dns}/{proxy}
  • Content Handling: Passthrough

Step 5: Setup Authentication

Configure AWS Cognito User Pool:

  1. Create Cognito User Pool for customer authentication
  2. Create App Client for API access
  3. Configure API Gateway Authorizer
  4. Apply authorizer to protected endpoints

Environment Variables

Set up the following environment variables:

API_GATEWAY_URL=https://your-api-id.execute-api.region.amazonaws.com/prod
COGNITO_USER_POOL_ID=your-user-pool-id
COGNITO_CLIENT_ID=your-client-id

Testing the API Gateway

Use the following curl commands to test endpoints:

# Test product listing (public endpoint)
curl -X GET "${API_GATEWAY_URL}/api/v1/products"

# Test order creation (authenticated endpoint)
curl -X POST "${API_GATEWAY_URL}/api/v1/orders" \
  -H "Authorization: Bearer ${ACCESS_TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{
    "customerId": "12345",
    "items": [
      {
        "productId": "prod-001",
        "quantity": 2,
        "price": 29.99
      }
    ]
  }'

Monitoring and Logging

Enable the following for API Gateway:

  • CloudWatch Logs: Detailed request/response logging
  • CloudWatch Metrics: Request count, latency, error rates
  • AWS X-Ray: Distributed tracing across microservices

Security Best Practices

  1. API Keys: For rate limiting and usage tracking
  2. WAF Integration: Protect against common attacks
  3. Resource-based Policies: Fine-grained access control
  4. Request Validation: Validate request parameters and body
  5. Response Caching: Improve performance and reduce backend load
